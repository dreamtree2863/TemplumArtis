# Templum Artis Music (모바일 PWA)

내 **Google Drive 음악 라이브러리**를 폰에서 스트리밍 재생하는 설치형 웹앱(PWA).
MP3 태그(USLT)에 심어둔 **싱크 가사**를 재생에 맞춰 표시한다. 앱스토어 없이 폰
브라우저로 열어 "홈 화면에 추가"하면 앱처럼 쓸 수 있고, 안드로이드·아이폰 모두 동작한다.

기능: Drive 음악 목록/검색 · 스트리밍 재생 · 앨범아트/메타데이터 · **싱크 가사** ·
플레이리스트(폰에 저장) · 잠금화면 컨트롤(미디어세션) · 셔플/반복.

---

## 1. Google OAuth 클라이언트 ID

이 앱은 서버 없이 브라우저에서 직접 Drive를 읽으므로 OAuth 클라이언트 ID가 필요하다.

> **이미 만들어 둔 게 있다:** 기존 **Templum Sapientiae Mobile** PWA의 클라이언트 ID가
> `app.js`에 기본값으로 들어가 있고 로그인 화면에도 미리 채워져 있다. 이 음악 PWA를
> **같은 GitHub Pages 계정(같은 `https://<아이디>.github.io` 출처)** 에 올리면 그 ID의
> 승인된 원본이 이미 등록돼 있어 **새로 만들 필요가 없다** — 바로 로그인하면 된다.
> 다른 도메인(Netlify 등)에 올릴 때만 Cloud 콘솔에서 그 출처를 추가하면 된다.

아래는 새로 만들 경우의 절차(최초 1회, 무료):

1. https://console.cloud.google.com → 우상단 프로젝트 선택 → **새 프로젝트** 생성
2. 좌측 **API 및 서비스 → 라이브러리** → **Google Drive API** 검색 → **사용 설정**
3. **API 및 서비스 → OAuth 동의 화면**
   - User Type: **외부** → 만들기
   - 앱 이름/이메일 대충 입력
   - **테스트 사용자**에 **본인 Gmail 계정 추가** (이게 없으면 로그인 차단됨)
   - 범위(scope)는 추가 안 해도 됨 (앱이 실행 시 요청)
4. **API 및 서비스 → 사용자 인증 정보 → 사용자 인증 정보 만들기 → OAuth 클라이언트 ID**
   - 애플리케이션 유형: **웹 애플리케이션**
   - **승인된 자바스크립트 원본**에 이 앱을 올릴 주소를 추가:
     - GitHub Pages면 `https://<본인아이디>.github.io`
     - 로컬 테스트면 `http://localhost:8000`
   - 만들기 → 나온 **클라이언트 ID**(`...apps.googleusercontent.com`)를 복사
5. 앱 첫 화면에 이 클라이언트 ID를 붙여넣고 로그인.

> 동의 화면이 "테스트" 모드라도 본인(테스트 사용자)은 그대로 쓸 수 있다. 로그인 때
> "확인되지 않은 앱" 경고가 나오면 **고급 → 이동**을 눌러 진행하면 된다.

## 2. 배포 (HTTPS 필요)

PWA·구글 로그인은 보안 컨텍스트(HTTPS 또는 localhost)에서만 동작한다. 가장 쉬운 무료 방법:

### GitHub Pages
1. GitHub에 새 저장소 생성 → 이 `mobile/` 폴더의 파일 전부 업로드(루트에)
2. 저장소 **Settings → Pages** → Source를 `main` 브랜치 `/root`로 설정
3. 몇 분 뒤 `https://<아이디>.github.io/<저장소>/` 로 접속 가능
4. 그 주소(정확히는 `https://<아이디>.github.io`)를 위 4번 **승인된 원본**에 넣기

Netlify / Cloudflare Pages / Vercel에 이 폴더를 드래그해도 된다(HTTPS 자동).

### 로컬에서 먼저 테스트 (PC)
```
cd mobile
python -m http.server 8000
```
→ PC 크롬에서 `http://localhost:8000` 접속. (승인된 원본에 `http://localhost:8000` 추가)

## 3. 폰에 설치

- **안드로이드(크롬)**: 앱 주소 접속 → 메뉴 → **홈 화면에 추가 / 앱 설치**
- **아이폰(사파리)**: 앱 주소 접속 → 공유 → **홈 화면에 추가**

설치하면 주소창 없이 전체화면 앱으로 실행된다.

## 한계 / 참고
- **검색 범위 = 지정한 음악 폴더(하위 포함)뿐**이다. Drive 전체를 뒤지지 않는다.
  기본값은 `Junho's Data/취미/음악`. 로그인 화면에서, 또는 로그인 후 상단 **📁** 버튼으로
  경로를 바꾸거나 여러 폴더(한 줄에 하나)를 추가할 수 있다. 다른 위치의 곡을 넣고 싶으면
  그 폴더 경로를 여기에 추가하면 된다.
- 곡은 **스트리밍 재생**한다: 서비스워커가 `<audio>`의 Drive 요청에 인증 헤더를 주입하고
  Range 요청을 그대로 전달해, 통째로 받지 않고 즉시 재생·탐색이 된다(Templum Sapientiae
  Mobile에서 검증된 방식). 메타/커버/가사는 파일 앞부분(태그 ~1MB)만 따로 받아 파싱한다.
- 목록은 파일명 기준으로 먼저 보이고, **재생하는 순간** 태그에서 정확한 제목/아티스트/
  커버/가사를 읽는다(2201곡 태그를 미리 다 받지 않기 위함).
- 라이브러리 목록은 폰에 캐시된다. 곡을 추가/변경했으면 앱에서 **⟳(새로고침)**.
- 데스크톱 앱과 **같은 Drive 파일**을 읽으므로, 거기서 심어둔 싱크 가사가 그대로 나온다.
